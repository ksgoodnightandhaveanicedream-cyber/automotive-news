# 自動車業界ニュースまとめアプリ 仕様書

## 1. 目的

自動車業界への転職活動に向けて、自動車業界全般および SDV（Software Defined Vehicle）関連の動向を効率よくキャッチアップするため、自動車関連ニュースサイトの最新記事を自動収集し、一覧で確認できる静的Webアプリを構築する。

このアプリは自動車業界専門の情報収集を目的とし、経済ニュース全般は扱わない。ただし総合経済メディアの中にも自動車関連の良質な記事はあるため、対象を自動車関連記事に絞ったうえで一部の経済メディアも収集対象に含める。

## 2. 想定利用シーン

- 毎朝/隔日など好きなタイミングでブラウザ（PC・スマートフォン）からサイトを開き、最新記事一覧を確認する。
- キーワード検索（例: SDV, 自動運転, EV, トヨタ 等）で関心テーマの記事を絞り込む。
- 気になった記事はリンクから元サイトに飛んで読む（本文の複製・再配布はしない）。

## 3. 全体構成

- **形態**: 静的サイトとしてGitHub Pagesにデプロイ（publicリポジトリ、誰でも閲覧可）
- **技術スタック**: Python（収集バッチ） + 静的HTML/CSS/JS（表示）
  - 収集: `feedparser`（RSS）。東洋経済オンラインのみRSSに自動車専用の配信がないため、タグ別記事一覧ページ（`/list/tag/自動車トレンド` 等）を直接取得する（robots.txtで許可された範囲）
  - 保存: JSONファイル（`docs/data/articles.json`）に記事メタデータを永続化・重複排除
  - 定期収集・更新: GitHub Actionsのscheduled workflow（例: 1時間ごと）+ `workflow_dispatch`による手動実行
  - 表示: `docs/`配下の静的HTML/CSS/JSがJSONを読み込んでレンダリング。検索はクライアントサイドJSで実装

```
[GitHub Actions (cron/手動)] --RSS--> [scripts/fetch_news.py]
        --> [docs/data/articles.json を更新してcommit&push]
        --> [GitHub Pages が docs/ を配信] --> [ブラウザ/スマホから閲覧]
```

補足: PCを起動していなくても、GitHub Actions側で収集・更新が行われ、GitHub Pagesは常時アクセス可能。スマートフォンからも通常のWebサイトとして閲覧できる。

## 4. 機能要件

### 4.1 記事収集
- 登録されたRSSフィードを巡回し、記事タイトル・URL・公開日時・出典サイト名を取得する。
- 取得済みのURLは重複登録しない（URLをユニークキーとしてJSONに保存）。
- 東洋経済オンラインについては、RSSに自動車専用の配信がないため、サイトのタグ別記事一覧ページ（`scripts/sources.py` の `type: "toyokeizai_tag"` / `tags: ["自動車トレンド"]` で指定）を直接取得し、そのタグに属する記事のタイトル・URL・公開日時を収集する。ページネーションを辿り、記事の公開日が30日より古くなった時点で打ち切る。
  - 検証の結果、東洋経済の全記事RSS（ビジネス・ライフ等を含む全ジャンル横断で直近20件のみ配信）から自動車関連記事を判定する方式では、直近20件という狭い範囲に自動車記事がほとんど含まれず月1件程度しか拾えなかった。タグ別記事一覧ページを直接見に行く方式に切り替えたことで、直近30日で約18件を収集できるようになった。
- 30日以上前の記事はデータから除外し、肥大化を防ぐ。
- ソース一覧から外したサイトの過去記事は次回収集時に自動的に削除される。

### 4.2 一覧表示画面
- 記事を公開日時の新しい順に一覧表示（タイトル、出典サイト名、公開日時、リンク）。
- キーワード検索（タイトルの部分一致）。
- 「既読」の簡易管理（クリックしたら既読扱いにする程度のシンプルなもの。ブラウザの`localStorage`で管理）。

### 4.3 更新
- GitHub Actionsのscheduled workflowで一定間隔（例: 1時間ごと）に自動収集・`articles.json`更新・GitHub Pagesへの反映を行う。
- GitHub Actionsの`workflow_dispatch`により、GitHub上の操作で即座に手動実行も可能。

## 5. データモデル（`docs/data/articles.json`）

```json
{
  "updated_at": "2026-07-22T10:30:00+09:00",
  "articles": [
    {
      "title": "トヨタ、新型SDVプラットフォームを発表",
      "url": "https://example.com/article/123",
      "source": "Car Watch",
      "published_at": "2026-07-22T10:30:00+09:00"
    }
  ]
}
```

- URLをユニークキーとして重複排除する。
- 「既読」管理はサーバー側に保存先がないため、ブラウザの`localStorage`にクリック済みURLを保持する簡易実装とする。

## 6. ニュースソース（`scripts/sources.py`）

自動車専門メディアはRSSをそのまま収集、総合経済メディア（東洋経済オンライン）はタグ別記事一覧ページを直接取得する。

| サイト名 | 種別 | 収集方法 | 備考 |
|---|---|---|---|
| Car Watch (Impress) | 自動車専門 | RSS: `http://car.watch.impress.co.jp/docs/car.rdf` | フィルタなし |
| carview! (Yahoo!ニュース経由) | 自動車専門 | RSS: `https://news.yahoo.co.jp/rss/media/carv/all.xml` | フィルタなし |
| webCG (Yahoo!ニュース経由) | 自動車専門 | RSS: `https://news.yahoo.co.jp/rss/media/webcg/all.xml` | フィルタなし |
| ベストカーWeb (Yahoo!ニュース経由) | 自動車専門 | RSS: `https://news.yahoo.co.jp/rss/media/bestcar/all.xml` | フィルタなし |
| 日刊自動車新聞 (Yahoo!ニュース経由) | 自動車専門 | RSS: `https://news.yahoo.co.jp/rss/media/netdenjd/all.xml` | フィルタなし |
| 東洋経済オンライン | 総合経済 | タグ一覧ページ: `https://toyokeizai.net/list/tag/自動車トレンド`（ページネーション対応） | 「自動車トレンド」タグの記事のみ |

> 将来的にソースを追加する場合も、上記表とRSS・利用規約を確認したうえで `scripts/sources.py` に追記する。東洋経済に他の自動車関連タグ（例: 自動車販売・購入）を追加したい場合は `tags` リストに追記するだけでよい。

## 8. 開発ステップ

1. ニュースソースの最終選定（無料・RSS確認済みのものからスタート）— 完了
2. 収集スクリプト（`scripts/fetch_news.py`）の実装 — 完了
3. 静的フロントエンド（`docs/index.html` 等）の実装 — 完了
4. GitHub Actions workflowの実装（定期実行 + 手動実行 + 自動commit&push）
5. GitHubリポジトリ作成・push・GitHub Pages有効化
6. 必要に応じてソース追加・UI改善

## 9. 非機能・留意事項

- 各サイトの著作権・利用規約を遵守し、記事本文は保存・表示せず、タイトル・リンク・出典・日時のみを扱う。
- publicリポジトリ・GitHub Pagesで公開するため、収集対象は無料で閲覧可能なニュースサイトに限定する。
- 東洋経済オンラインのタグ一覧ページ取得は `robots.txt` で許可された範囲（`/list/tag/`配下は許可、`/list/search`等のみ禁止）。1タグあたり最大6ページまでという上限を設け、直近30日分を取得できた時点でページネーションを打ち切る。GitHub Actionsは1時間に1回のみの実行であり、過度なアクセスにはならない。他サイトへのスクレイピングを追加する場合も同様に `robots.txt` および利用規約を事前に確認する。
